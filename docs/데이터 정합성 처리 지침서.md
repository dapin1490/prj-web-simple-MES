## 심플 MES 데이터 정합성 처리 최종 지침서

### 1. 목적 및 범위

- 목적: 서로 다른 소스에서 수집된 수주 데이터(에스윈텍)와 공정 데이터(의류 염색) 간의 수치적 모순을 제거하고, 실측 기반의 정합성 있는 데모 데이터셋을 구축함.
- 범위: 2022년 1월부터 10월까지의 의류 공정 실측 기간에 해당하는 약 231일간의 데이터.

### 2. 수치 불일치 해결: 실측 기반 덮어쓰기 전략

데이터의 신뢰성을 확보하기 위해 실제 생산된 양을 주문량으로 간주하는 하향식(Bottom-up) 동기화 방식을 적용합니다.

| 단계 | 처리 공정 | 적용 방법 | 관련 데이터 속성 |
| --- | --- | --- | --- |
| 1단계 | 생산량 집계 | 의류 공정 데이터를 날짜별로 그룹화하여 총 생산 합계를 산출함. | `염색 가동 길이` , `INSRT_DT` |
| 2단계 | 주문량 동기화 | 산출된 일일 생산 합계값을 에스윈텍 주문량 필드에 덮어씀. | `day_order` |
| 3단계 | 날짜 치환 | 에스윈텍의 주문 날짜를 의류 공정의 실측 날짜로 변경함. | `date` , `order_date` |
| 4단계 | 데이터 필터링 | 실측 로그가 없는 기간의 주문 기록은 모두 삭제하여 정합성을 유지함. | (전체 기간 231일 기준) |

### 3. 제품 정보 재구성 (Private Data 해제)

비공개 처리된 제품 정보를 의류 공정의 특성에 맞춰 다음과 같이 복원합니다.

- 식별자 매핑: 에스윈텍의 제품 `code`를 의류 공정의 `JOB_CD`("F12-1203")로 치환함.
- 제품명 부여: 비공개된 `name` 속성을 공정 성격에 맞춰 "고신축 폴리에스터 원단", "기능성 나일론" 등으로 임의 명명함.
- 계층 구조: 에스윈텍의 `hierarchy` 정보를 활용하여 상위 제품군 분류 체계를 유지함.

### 4. 품질 데이터 연계 로직

실측 데이터의 한계를 고려하여 품질 지표를 다음과 같이 연결합니다.

- 품질 매핑: 전체 2,127개 로트중 품질 측정값이 존재하는 434개 로트에 대해서만 `염색 색차 DE` 값을 연결함.
- 판정 기준: 종합 색차(DE) 값이 1.0 미만일 경우 '양호', 1.0 이상일 경우 '주의' 혹은 '불량'으로 판정하는 비즈니스 로직을 적용함.

### 5. 데이터베이스 정합성 검증 규칙 (Validation Rules)

- 총량 일치: `SalesOrders.order_qty` = $\sum(\text{WorkOrders.planned\_qty})$ 관계가 반드시 성립해야 함.
- 시간 선후 관계: `SalesOrders.order_date` $\le$ `ProductionLogs.timestamp` 순서를 준수해야 함.
- 참조 무결성: 모든 `ProductionLogs`는 유효한 `wo_id`(로트 번호)를 참조해야 함.
